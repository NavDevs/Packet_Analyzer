What is DPI?

Deep Packet Inspection (DPI) is an advanced network technology that examines the actual content of data packets, not just headers.

📊

Packet Analysis

Examines every packet's payload to identify the actual application generating the traffic, not just port numbers.

🚫

Traffic Blocking

Create custom rules to block specific applications, domains, or IP addresses based on your requirements.

📈

Insights & Reports

Generate detailed reports showing traffic breakdown by application, protocol, and destination.

How DPI Differs from Regular Firewalls

Traditional Firewall

Only looks at packet headers (IP addresses, port numbers). Can block by port 80 or IP range, but can't tell if traffic is YouTube vs Netflix.

DPI Engine

Looks inside the packet payload. Identifies the actual application by examining protocol signatures, SNI fields, and content patterns.

Technical Details

Universal packet inspection capabilities

Supported Protocols

TLS/SSL (HTTPS)

Extracts Server Name Indication (SNI) from TLS handshakes to identify websites even with encryption.

HTTP

Analyzes Host headers and URL paths to identify web applications and services.

QUIC

Identifies modern protocols used by Google services including YouTube and Gmail.

DNS

Inspects DNS queries to identify domain lookups and potential tunneling attempts.

Server Name Indication (SNI) is a TLS extension that allows a client to specify which hostname it's trying to connect to during the SSL handshake.

When you visit a website, your browser sends the domain name (e.g., "youtube.com") in cleartext during the initial TLS handshake—even before encryption is established.

DPI extracts this SNI field to determine which website or service the encrypted connection is for, without decrypting the actual content.

PCAP (Packet Capture) is a standard file format for storing network traffic data. It's like a recording of all network packets that passed through a network interface.

How to create a PCAP:

  • Wireshark: File → Capture → Start, then save as .pcap
  • tcpdump: tcpdump -w capture.pcap
  • Linux: sudo tcpdump -i eth0 -w traffic.pcap

PCAP files contain the complete packet data including headers and payload, allowing DPI to perform deep analysis.

When the DPI engine processes packets, it applies your blocking rules in real-time:

  1. App Blocking: If a packet's SNI matches a known app pattern (e.g., "googlevideo.com" = YouTube), it can be dropped.
  2. Domain Blocking: Exact domain matches in SNI or DNS queries trigger packet drops.
  3. IP Blocking: Source/destination IP addresses are checked against blocklists.

Dropped packets are excluded from the output PCAP, simulating a firewall in action.

The DPI engine uses a multi-threaded pipeline for high performance:

  • Load Balancers: Distribute packets across worker threads
  • Worker Threads: Each processes packets through the full pipeline
  • Pipeline Stages: Packet Parser → SNI Extractor → Rule Engine → Output

This architecture allows processing millions of packets efficiently.
